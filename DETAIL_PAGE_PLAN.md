# FIFA 详情页规划

> 说明：本文件用于定义 `比赛详情页` 与 `玩法详情页` 的入口规则、默认选中规则、布局结构与内容模块。后续详情页开发应以本文件为准，并与 [HOME_PAGE_PLAN.md](/Users/daxiong/Desktop/workSpace/FIFA/HOME_PAGE_PLAN.md:1) 和 [MATCH_ORDER_INTERACTION_RULES.md](/Users/daxiong/Desktop/workSpace/FIFA/MATCH_ORDER_INTERACTION_RULES.md:1) 保持一致。

## 1. 详情页目标

- 承接首页 `比赛` 与 `玩法` 的进一步信息展开。
- 承接首页中未展示完整的盘口、候选项与细分玩法。
- 保持 `详情内容 + 下单区` 的信息结构。
- 保证“从列表进入详情页时”的默认选中状态可预测且可复现。

## 1.1 详情页尺寸规则

- 详情页中的字号、按钮、输入框、标签栏、市场卡片与行高，统一按 `移动端优先` 的方式设计。
- 默认样式先满足小屏手机可读性与可点击性，再在更大断点上适度放大。
- 详情页不能以桌面端为主设计后再反向压缩移动端。

## 2. 入口规则

### 2.1 比赛区进入详情页

- 首页比赛列表中，每场比赛卡片的 `非按钮区域` 均可点击进入详情页。
- `非按钮区域` 包括：
  - 时间与交易量区域
  - 队伍信息区域
  - 比分区域
  - 其他非盘口按钮的卡片主体空白区
- 卡片内已有明确交互的按钮不复用跳转：
  - `胜负线 / 让分 / 总分` 盘口按钮
  - `比赛视图` 按钮
- 点击这些按钮时，优先执行当前按钮自身的下单或切换逻辑，而不是跳转。

### 2.2 玩法区进入详情页

- 首页玩法卡片的 `非按钮区域` 可点击进入详情页。
- 列表型玩法卡片中，如果用户点击的是某一条候选项所在行，应携带该候选项作为预选结果进入详情页。
- 如果用户点击的是卡片空白区、标题区、图标区或其他非候选项区域，则进入详情页后不带显式候选项来源。

### 2.3 进入详情时的默认选中规则

- 比赛详情页：
  - 如果进入前没有显式带入某个盘口选择，则默认选中该比赛的第一个可交易项。
  - 当前约定的默认项为该比赛 `胜负线` 的第一项。
- 玩法详情页：
  - 如果进入前携带了候选项来源，则详情页和下单区默认预选该候选项。
  - 如果进入前没有携带候选项来源，则默认选中该详情页数据中的第一项。

## 3. 首页默认选中规则

- 首页 `比赛` Tab 只要存在比赛数据，桌面端右侧悬浮下单区应默认选中第一场比赛的第一个可交易项。
- 该默认选中仅用于桌面端右侧固定下单区的初始展示。
- 移动端首页不因为默认选中而自动弹出下单抽屉。
- 如果用户已经主动切换过选中项，则后续页面内刷新状态不应无条件覆盖用户当前选择。

## 4. 路由建议

- 比赛详情页建议路由：
  - `/matches/:matchId`
- 玩法详情页建议路由：
  - `/markets/:marketId`

建议的参数承载方式：

- 显式预选项可通过 query 或路由 state 传递。
- 例如：
  - `/markets/world-cup-winner?candidate=spain`
  - 或通过 router state 写入 `preselectedCandidateId`

## 5. 比赛详情页布局

### 5.1 页面整体结构

比赛详情页应采用以下一级结构：

1. 顶部比赛信息区
2. 内容标签切换区
3. 主内容区
4. 右侧悬浮下单区

其中：

- `1024px` 及以上：
  - 使用 `左侧详情内容 + 右侧悬浮下单区` 的双栏布局
- `1024px` 以下：
  - 内容区单列展开
  - 下单区继续沿用移动端抽屉 / 底部弹层逻辑

### 5.2 顶部比赛信息区

顶部信息区应严格参考截图 1 的信息层级，包含：

- 比赛标题：
  - 例如 `Mexico vs South Africa`
- 开赛倒计时或时间提示
- 中间分隔线与赛事来源样式占位
- 主队国旗与名称
- 客队国旗与名称
- 中央比赛时间与日期

实现要求：

- 顶部区域是详情页的上下文锚点，不承担交易操作。
- 队伍信息、时间信息与标题信息要稳定对齐。
- 长队伍名称在必要时可换行，但不应压坏整体三栏结构。

## 6. 比赛详情页内容标签

比赛详情页内容区需要提供以下三个标签：

1. `比赛盘口`
2. `准确比分`
3. `半场结果`

规则：

- 默认进入详情页时，先落在 `比赛盘口`
- 标签切换只影响左侧内容区
- 右侧悬浮下单区继续保留，并跟随当前内容区中的点击操作联动

## 7. 比赛盘口标签页

该标签页的结构参考截图 2。

### 7.1 市场分组

至少包含以下模块：

- `Moneyline`
- `让分`
- `总分`
- `Both Teams to Score?`

### 7.2 每个市场模块的公共结构

每个市场区块包含：

1. 左侧市场名称
2. 左侧交易量
3. 右侧可点击盘口按钮
4. 如果该市场有 line 维度，则在下方展示 line selector

### 7.3 盘口按钮规则

- `Moneyline`：
  - 显示三项结果按钮
  - 例如 `MEX / DRAW / RSA`
- `让分`：
  - 显示当前选中 line 对应的一对盘口
  - 例如 `MEX -1.5 / RSA +1.5`
  - 下方展示 line selector
- `总分`：
  - 显示当前选中 line 对应的一对盘口
  - 例如 `O 2.5 / U 2.5`
  - 下方展示 line selector
- `Both Teams to Score?`：
  - 显示 `YES / NO`
  - 无 line selector

### 7.4 line selector 规则

- `让分` 与 `总分` 均需要在详情页中展示 line selector。
- `让分` 可出现重复数值，因为方向不同。
- `总分` 使用单纯数值线。
- 当前选中的 line 需要有高亮和指示标记。
- 点击 line 后：
  - 区块内盘口按钮更新
  - 悬浮下单区同步更新

## 8. 准确比分标签页

该标签页结构参考截图 3。

### 8.1 展示形式

- 使用纵向列表承载多个精确比分条目。
- 每个条目单独成卡。

### 8.2 单条结构

每一条准确比分项包含：

1. 标题
  - 例如 `Exact Score: 0-0`
2. 交易量
3. 右侧 `YES`
4. 右侧 `NO`

### 8.3 交互规则

- 点击 `YES` 或 `NO` 后，右侧悬浮下单区进入该比分命题的下单状态。
- 该标签页不需要额外的 line selector。
- 条目数量可较多，应支持长列表滚动。

## 9. 半场结果标签页

该标签页结构参考截图 4。

### 9.1 模块结构

- 顶部延续详情页内容标签。
- 内容区至少包含一个 `Halftime Result` 区块。

### 9.2 区块内容

区块内展示：

1. 市场名称
2. 交易量
3. 三项盘口按钮
  - 主队
  - 平局
  - 客队

### 9.3 联动规则

- 点击任一半场结果项，右侧悬浮下单区切换到当前半场结果命题。
- 它与全场 `Moneyline` 属于不同命题，不能共享选中态。

## 10. 玩法详情页规则

### 10.1 页面定位

- 玩法详情页承接首页玩法卡片的完整信息。
- 首页只展示预览，详情页展示完整候选列表或完整问题信息。

### 10.2 列表型玩法详情页

- 若玩法存在多候选项，详情页中应完整展示所有候选对象。
- 首页只显示最多 2 条，剩余条目在详情页中继续承接。
- 如果用户从首页点击了某一候选项进入，则详情页默认预选该项并同步下单区。
- 如果无显式候选项来源，则默认预选第一项。

### 10.3 二元玩法详情页

- 若玩法本身是一个单命题二元市场，则详情页展示完整问题内容与 `Yes / No` 操作。
- 如果首页无显式候选项来源，则详情页直接落在该命题本身。

## 11. 悬浮下单区在详情页中的规则

- 桌面端详情页继续保留右侧悬浮下单卡片。
- 移动端详情页继续使用抽屉 / 底部弹层，不显示固定右栏。
- 下单区数据来源继续使用全局状态。

详情页进入时的默认态：

- 比赛详情：
  - 默认选中第一项，除非入口已带显式盘口上下文
- 玩法详情：
  - 默认使用入口预选项
  - 若无入口预选项，则默认第一项

## 12. 实现建议

建议按以下顺序落地：

1. 补充路由：
   - `matches/:matchId`
   - `markets/:marketId`
2. 补充详情页数据模型：
   - 比赛详情市场数据
   - 准确比分数据
   - 半场结果数据
   - 玩法详情完整候选列表
3. 补充全局状态：
   - `selectedDetailContext`
   - `preselectedCandidateId`
   - `entrySource`
4. 完成首页到详情页跳转
5. 完成详情页默认选中与右侧下单联动
